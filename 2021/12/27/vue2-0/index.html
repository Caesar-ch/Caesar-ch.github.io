    条件渲染指令

    v-if

    ​ 动态创建和移除元素,设置flag可动态删除，

    ​ v-else-if

    ​ v-else

    ​ v-if和v-else要写在同级标签下

    v-if可以单独使用，可以使用两个v-if，达到v-if和v-else的效果

    v-show

    ​ 动态添加和移除display：none样式，是css样式的动态切换

    • 实现方式: v-if是动态的向DOM树内添加或者删除DOM元素，v-show是通过设置DOM元素的display样式属性控制显隐。
    • 编译过程: v-if切换有一个局部编译卸载的过程，切换过程中合适地销毁和重建内部的事件监听和子组件，v-show只是简单的基于CSS切换。
    • 编译条件: v-if是惰性的，如果初始条件为假，则什么也不做，只有在条件第一次变为真时才开始局部编译， v-show是在任何条件下都被编译，然后被缓存，而且DOM元素保留。
    • 性能消耗: v-if有更高的切换消耗，v-show有更高的初始渲染消耗。
    • 使用场景: v-if适合条件不太可能改变的情况，v-show适合条件频繁切换的情况

    props自定义 属性

    实现了父组件（父->子）向子组件传值

    • ​ **1.**先在子组件中，通过props来自定义属性

    • ​ **2.**父组件中，负责把数据，通过v-bind：绑定给子组件

      ​ 极大的提高组件的复用性！

      ​ 定义props有两种格式

      ​ 数据格式：

      ​ 简单

      ​ props：[‘int’]

      ​ 对象格式：

      ​ props：{

      ​ init：{

      ​ type：Number，

      ​ default：0，

      ​ required：true

      ​ }

      ​ }

      ​ props是只读的，因此在项目开发中，不要直接修改它的值。控制台会报err

      ​ 可以吧props的值，“转存”到data中，就可以进行读写操作

    生命周期函数

    ​ 概念：在生命周期的不同阶段，会按次序、自动依次执行的函数，叫做生命周期函数。强调的是时间点。

    组件之间的数据共享

    子->父

    ​ 自定义事件：

    ​ 在子组件中调用this.$emit()来触发自定义事件：

    ​ 参数1：字符串，表示自定义事件的名称

    ​ 参数2：值，要发送给父组件的数据

    ​ 在父组件中，通过v-on：来绑定自定义事件，并提供一个事件处理函数。通过事件处理函数的形参，来接收到子组件传递过来的数据

    ref获取dom元素

    ​ 1.通过给template模板里面标签设置 ref=‘特定类名’
    ​ 2.在 methods 定义函数 通过 this.$refs.特定类名拿到dom元素
    ​ 3.修改this.$refs.特定类名.style.属性名=’属性值’，操作dom元素

    ​ ****ref优于子组件向父组件传值的操作****

    ref获取组件实例(即可获得组件的this以调用组件方法数据)
    1.给组件设置ref=‘特定类名的值’
    2.在methods里定义函数，通过this.$refs.’特定类名’，拿到了子组件的实例this

    ​ （ 动态会涉及生命周期，渲染完毕的问题，需要在dom重新渲染成功后获取dom元素，进行操作，需要延时获取this.$nextTick(回调函数) ）

    ​ 3.此时便可以通过这个this拿到子组件的方法和数据，进行操作
